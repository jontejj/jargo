Package se.softhouse.jargo Description

An argument and options parser for java

Glossary

  • Option a simple flag parameter such as "-l", created with Arguments.optionArgument(String, String...). Doesn't accept any Parameter
  • Parameter the second part of an Argument, parsed with a StringParser
  • Argument both the name and the parameter, such as "--numbers 1,2,3"
  • Named Argument an argument with a name such as "--numbers"
  • Indexed Argument an argument without a name, such as "file_to_build" in "buildProgram file_to_build"
  • CommandLineParser Understands how to manage several Arguments
Examples

 See CommandLineParser, Arguments, ArgumentBuilder for how to use this API
 
Localization

 Arguments.integerArgument(String...), Arguments.longArgument(String...) and so on,
 formats and parses numbers in a Locale sensitive way.
 
API compatibility notes

 Public methods that have Guava types in their method signature such as:
 ArgumentBuilder.limitTo(com.google.common.base.Predicate)
 ArgumentBuilder.defaultValueSupplier(com.google.common.base.Supplier)
 StringParsers.asFunction(StringParser)
 May be changed when JDK 8 is here as it will contain those interfaces without the need of Guava. For now they are marked with Beta

Inspiration (a.k.a Credits)

If you recognize any features in this API it may come from:
  • git - the Command feature, suggestions for invalid arguments
  • maven - executing several commands
  • jcommander - almost everything else (this is what I based the feature set on)
