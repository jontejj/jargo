se.softhouse.jargo

Class CommandLineParser

  • java.lang.Object
    • se.softhouse.jargo.CommandLineParser

  • @ThreadSafe
public final class CommandLineParser
extends Object
    Manages multiple Arguments and/or Commands. The brain of this API. Its primary goal is to decide which Argument each input string belongs to. A CommandLineParser can be reused to parse arguments over and over again (even concurrently). Different CommandLineParsers can even share Argument configurations as Argument instances are immutable as well. Documentation through example: 
     
 import static se.softhouse.jargo.Arguments.*;
 ...
 String[] args = {"--enable-logging", "--listen-port", "8090", "Hello"};
 
 Argument<?> helpArgument = helpArgument("-h", "--help"); //Will throw when -h is encountered
 Argument<Boolean> enableLogging = optionArgument("-l", "--enable-logging").description("Output debug information to standard out").build();
 Argument<String> greetingPhrase = stringArgument().description("A greeting phrase to greet new connections with").build();
 Argument<List<Integer>> ports = integerArgument("-p", "--listen-port")
                                                .defaultValue(8080)
                                                .description("The port clients should connect to.")
                                                .metaDescription("<port>")
                                                .limitTo(Range.closed(0, 65536))
                                                .repeated().build();
 
 try
 {
   ParsedArguments arguments = CommandLineParser.withArguments(helpArgument, greetingPhrase, enableLogging, ports).parse(args);
   assertThat(arguments.get(enableLogging)).isTrue();
   assertThat(arguments.get(ports)).isEqualTo(Arrays.asList(8090));
   assertThat(arguments.get(greetingPhrase)).isEqualTo("Hello");
 }
 catch(ArgumentException exception)
 {
   System.out.println(exception.getMessageAndUsage());
   System.exit(1);
 }
 
 
     For this program the usage would look like ("YourProgramName" is fetched from stack traces by default):
 
 Usage: YourProgramName [Arguments]
 
 Arguments:
 <string>                       A greeting phrase to greet new connections with
                                <string>: any string
                                Default:
 -l, --enable-logging           Output debug information to standard out
                                Default: disabled
 -p, --listen-port <port>       The port clients should connect to [Supports Multiple occurrences]
                                port: [0â€¥65536]
                                Default: 8080
 
 If something goes wrong during the parsing (Missing required arguments, Unexpected arguments, Invalid values),
 it will be described by the ArgumentException. Use ArgumentException.getMessageAndUsage() if you
 want to explain what went wrong to the user.
    Internationalization By default Locale.US is used for parsing strings and printing usages. To change this use locale(Locale).
    Thread safety concerns: If there is a parsing occurring while any modifying method is invoked, such as programName(String), the changes won't be seen until the start of the next parse invocation. Also, if the goal is to set programName(String) and programDescription(String) atomically you'll have to synchronize such changes externally.

      • withCommands

        public static <E extends Enum<E> & Runnable & Describable> CommandLineParser withCommands(Class<E> commandEnum)
        An alternative to Command that accepts an Enum following the Command Pattern. This alternative is viable if the commands don't accept any parameters. Example enum: 
         
 public enum Service implements Runnable, Describable
 {
   START{
           @Override
           public void run(){
             //Start service here
           }
 
           @Override
           public String description(){
             return "Starts the service";
           }
   };
 }
        The name for each command will be the enum constants name in lower case.
        Parameters:
        commandEnum - the Class literal for Service in the example above
        Returns:
        a CommandLineParser which you can call parse(String...) on
